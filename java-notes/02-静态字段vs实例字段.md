# static vs 非 static · 字段和方法属于谁

> 📍 来源：单例 V1 饿汉式
> 🎯 回答的核心问题："**同一个类里，为什么有些字段所有对象共享，有些每个对象一份？**"

---

## 🤔 场景问题

单例 V1 饿汉式里：

```java
class EagerPrinter {
    private static final EagerPrinter INSTANCE = new EagerPrinter();   // 🔵 static
    private int count = 0;                                              // 🟢 非 static
}
```

疑问：为什么 `INSTANCE` 是 `static`，`count` 不是？两者有什么本质区别？

---

## 🧠 核心一句话

> **`static` 字段属于"类"，非 `static` 字段属于"对象"。**
>
> 整个 JVM 里，一个类**只有一份** static 字段；但每 `new` 一个对象，就有一份独立的非 static 字段。

---

## 📊 内存视角看区别

```java
class User {
    static int totalUsers = 0;   // 类变量
    String name;                  // 实例变量
}

User u1 = new User();   u1.name = "Alice";
User u2 = new User();   u2.name = "Bob";
User.totalUsers = 100;
```

内存结构：

```
【方法区 / 元空间】
┌────────────────────────────┐
│  class User                │
│  ├── totalUsers = 100     │  ← 所有对象共享这一份
│  └── (方法字节码)          │
└────────────────────────────┘

【堆】
┌────────────┐      ┌────────────┐
│  u1 对象    │      │  u2 对象    │
│  name="Alice"│    │  name="Bob" │ ← 各自一份
└────────────┘      └────────────┘
```

**规律**：
- static 字段 → 存在**方法区（元空间）**，跟着"类"
- 实例字段 → 存在**堆**，跟着每个"对象"

---

## 📌 static 的用法总结

### 1. static 字段（类变量）
```java
class Counter {
    static int total = 0;
}

Counter.total;   // 通过类名访问
```

### 2. static 方法（类方法）
```java
class Math {
    public static int max(int a, int b) { return a > b ? a : b; }
}

Math.max(1, 2);   // 不需要 new Math()
```

**注意**：static 方法**不能直接访问非 static 字段**。因为 static 方法调用时可能根本没对象，哪来的实例字段？

### 3. static 代码块
```java
class Config {
    static Map<String, String> cache;
    static {
        cache = new HashMap<>();
        cache.put("version", "1.0");
    }
}
```
**类加载初始化阶段**执行，且**只执行一次**。

### 4. static 内部类
不依赖外部类实例（详见 [05-Java 内部类](./05-Java内部类.md)）。

## 🎯 回到单例 V1

```java
class EagerPrinter {
    private static final EagerPrinter INSTANCE = new EagerPrinter();
    // 👆 static 意味着整个 JVM 只有一份 INSTANCE，完美符合单例语义

    private int count = 0;
    // 👆 非 static，属于"那个唯一实例"的独有状态
}
```

**逻辑链**：
- INSTANCE 是 static → 只有一份 → 单例成立
- count 是实例字段 → 属于 INSTANCE 这个对象 → 所有用户共享累加（因为大家拿到的都是同一个 INSTANCE）

## ⚠️ 常见误区

### 误区 1："static 字段是全局变量"
**更精确的说**：static 字段是**类作用域**的字段，可以是 private（不全局）。
但 public static 字段**确实等价于全局变量**，且同样有全局变量的所有坏处。

### 误区 2："static 字段线程不安全"
**不是**。static 和并发安全**没关系**。关键看你是否存在并发修改。
- 多线程只读 `static final` 字段 → 安全
- 多线程修改 `static int count` → 不安全（需要加锁或用 AtomicInteger）

### 误区 3："static 方法不能被重写"
**对，但要区分**：
- static 方法**不参与多态**，不能被"覆写"（override）
- 但**可以被"隐藏"（hide）**：子类定义同名 static 方法，通过**编译期类型**决定调哪个

```java
class Parent {
    static void foo() { print("parent"); }
}
class Child extends Parent {
    static void foo() { print("child"); }
}

Parent p = new Child();
p.foo();          // 输出 "parent"！按编译类型
```

### 误区 4："能在 static 方法里用 this？"
**不能**。static 方法调用时没有"当前对象"，`this` 无意义，编译报错。

## 🎁 一个判断技巧

问自己：**"这个属性/行为，是属于'这一类事物整体'，还是属于'某个具体的个体'？"**

| 描述 | 属于 |
|------|------|
| 办公室打印机一共有几台 | 类 → static |
| 某台打印机的打印计数 | 对象（如果多台） |
| Math 里的 π 常数 | 类 → static |
| 一个用户的姓名 | 对象 |
| Logger 的输出文件路径（全程序一个） | 类 → static |
| 一个订单的下单时间 | 对象 |

---

## 💡 两条黄金规则

1. **能不用 static 就不用**。实例方法更灵活，便于单元测试和多态。
2. **static 一旦有可变状态，就要面对并发问题**。`static int count` 这种是并发 bug 的高发区。

---

## 📌 一句话总结

> **static 跟"类"走，非 static 跟"对象"走。单例用 static 字段存唯一实例，是这个机制最常见的应用。**
