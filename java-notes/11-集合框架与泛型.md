# Java 集合框架 & 泛型入门

> 📍 来源：建造者课 · `private List<String> vegetables = new ArrayList<>();`
> 🎯 回答的核心问题："**List 和 ArrayList 什么关系？`<String>` 是啥？为啥声明用接口、实例化用具体类？**"

---

## 🤔 场景问题

Builder 课的这一行代码：

```java
private List<String> vegetables = new ArrayList<>();
```

藏了四个知识点：
1. `List` 和 `ArrayList` 是什么关系
2. `<String>` 是什么（泛型）
3. `<>` 是什么（菱形运算符）
4. 为什么不直接 `ArrayList<String> list = new ArrayList<>();`

---

## 🧠 核心结论

> **Java 集合框架 = 接口定义"能做什么" + 实现类定义"怎么做"**。
>
> **泛型** = 给容器加一个类型参数，让编译器帮你管类型安全。
>
> **"面向接口编程"** = 声明用接口（List），实例化用具体类（ArrayList）。

## 📦 List 家族：**有序、可重复**

List 是最常用的集合类型，用来存"**有顺序可重复的元素序列**"。

### `ArrayList` —— 动态数组（90% 场景用它）

```java
List<String> list = new ArrayList<>();
list.add("a");
list.add("b");
list.add("a");   // 可以重复
// list: [a, b, a]
```

**底层**：`Object[]` 数组，自动扩容（通常扩容为 1.5 倍）。

**擅长**：
- ✅ 随机访问 `get(i)` — O(1)
- ✅ 末尾添加 `add(e)` — 平均 O(1)
- ✅ 遍历

**不擅长**：
- ❌ 中间插入 `add(i, e)` — O(n)（要后移元素）
- ❌ 按值删除 `remove(e)` — O(n)

### `LinkedList` —— 双向链表

```java
List<String> queue = new LinkedList<>();
queue.addFirst("a");      // 首部添加
queue.addLast("b");       // 尾部添加
```

**底层**：双向链表节点。

**擅长**：
- ✅ 首尾增删 — O(1)
- ✅ 中间插入（已定位节点时）— O(1)

**不擅长**：
- ❌ 随机访问 — O(n)（从头/尾遍历到第 i 个）

**实战建议**：**生产环境几乎只用 ArrayList**。LinkedList 的理论优势在实际场景中很难发挥（CPU 缓存对 ArrayList 更友好）。

### `Vector` —— 已过时

Java 1.0 的遗产，自带 `synchronized`，线程安全但性能差。**永远别用**。

### `CopyOnWriteArrayList` —— 并发安全

多线程读多写少场景用。内部 `volatile Object[]`，写时复制。

## 🎭 Set 家族：**无序、不重复**

存"**不允许重复**"的元素。

### `HashSet` —— 哈希集合（最常用）

```java
Set<String> set = new HashSet<>();
set.add("a");
set.add("b");
set.add("a");   // 重复，被忽略
// set: [a, b]（顺序不保证）
```

**底层**：内部其实是一个 `HashMap`，元素当 key，value 是占位对象。

**特点**：
- ✅ `contains(e)` — 平均 O(1)
- ❌ **无序**（遍历顺序不等于插入顺序）

### `LinkedHashSet` —— 保持插入顺序

```java
Set<String> set = new LinkedHashSet<>();
set.add("b"); set.add("a"); set.add("c");
// 遍历顺序：b, a, c（按插入顺序）
```

### `TreeSet` —— 按自然顺序排序

```java
Set<Integer> set = new TreeSet<>();
set.add(3); set.add(1); set.add(2);
// 遍历顺序：1, 2, 3（自动排序）
```

**底层**：红黑树。

---

## 🗂 Map 家族：**键值对**

`Map` 不继承 `Collection`，是独立的一支。存"**key → value**"映射。

### `HashMap` —— 最常用

```java
Map<String, Integer> map = new HashMap<>();
map.put("a", 1);
map.put("b", 2);
int n = map.get("a");    // 1
```

**底层**：数组 + 链表 + 红黑树（Java 8+ 链表超过 8 转红黑树）。

**特点**：
- ✅ `get` / `put` 平均 O(1)
- ❌ **无序**

### `LinkedHashMap` —— 保持插入顺序

```java
Map<String, Integer> map = new LinkedHashMap<>();
map.put("c", 3); map.put("a", 1); map.put("b", 2);
// 遍历顺序：c, a, b
```

### `TreeMap` —— 按 key 排序

```java
Map<String, Integer> map = new TreeMap<>();
map.put("banana", 2); map.put("apple", 1); map.put("cherry", 3);
// 遍历顺序：apple, banana, cherry（字典序）
```

### `ConcurrentHashMap` —— 并发安全

多线程读写场景的标配。内部分段锁（Java 8+ 改为 CAS + synchronized 单节点）。

## 📐 泛型（Generics）详解

### 什么是泛型

**"给类加一个类型参数"**，让同一份代码能处理不同类型。

```java
// 没有泛型（Java 1.4 之前）
List list = new ArrayList();
list.add("hello");
list.add(123);                        // 装进去的时候啥都行
String s = (String) list.get(0);      // 取出来要强转，容易 ClassCastException

// 有泛型（Java 5+）
List<String> list = new ArrayList<>();
list.add("hello");
list.add(123);                        // ❌ 编译错误
String s = list.get(0);               // 不用强转
```

**核心价值**：
- ✅ 编译期类型安全
- ✅ 不用强转
- ✅ 代码意图清晰

### 类型参数命名约定

| 字母 | 含义 | 用在 |
|------|------|------|
| `T` | Type | 通用类型 |
| `E` | Element | 集合元素 |
| `K`, `V` | Key, Value | Map |
| `R` | Result | 返回值 |
| `N` | Number | 数字 |

### 菱形运算符 `<>`（Java 7+）

```java
// Java 7 之前：
List<String> list = new ArrayList<String>();

// Java 7+：
List<String> list = new ArrayList<>();
//                              ↑
//                       菱形：自动从左推出 String
```

纯语法糖，功能一样。

### 类型擦除（Type Erasure）

**重要但少见坑**：Java 的泛型**运行时不存在**！编译后都擦除成 `Object`：

```java
List<String> list = new ArrayList<>();
list.getClass() == new ArrayList<Integer>().getClass();   // true！同一个 Class 对象
```

带来的限制：
- 不能 `new T[]`（数组要运行时知道类型）
- 不能 `if (x instanceof List<String>)`（运行时泛型信息没了）
- 不能重载 `foo(List<String>)` 和 `foo(List<Integer>)`（擦除后方法签名一样）

### 通配符 `? extends` / `? super`（PECS 原则）

```java
// ? extends T：可读不可写（"生产者"）
List<? extends Number> producer = new ArrayList<Integer>();
Number n = producer.get(0);      // ✅ 能读
// producer.add(1);              // ❌ 不能写（不知道具体是 Integer 还是 Double）

// ? super T：可写不可读（"消费者"）
List<? super Integer> consumer = new ArrayList<Number>();
consumer.add(1);                 // ✅ 能写 Integer
// Integer n = consumer.get(0);  // ❌ 只能读成 Object
```

**PECS 原则**：**P**roducer **E**xtends, **C**onsumer **S**uper。（记不住也没关系，用到 API 设计时再回头看。）

## 🎯 "面向接口编程" —— 为什么声明用 List

### ❌ 不推荐
```java
ArrayList<String> list = new ArrayList<>();
```

### ✅ 推荐
```java
List<String> list = new ArrayList<>();
```

**好处**：

1. **可替换**：想换实现类不影响调用方
   ```java
   List<String> list = new ArrayList<>();
   list = new LinkedList<>();   // ✅ 可以无缝替换
   ```
2. **接口更宽松**：方法参数接受任何 List 实现
   ```java
   void process(List<String> list) {}
   process(new ArrayList<>());    // ✅
   process(new LinkedList<>());   // ✅
   process(List.of("a", "b"));    // ✅
   ```
3. **契约导向**：调用方不关心具体实现，只关心"能做什么"

**铁律**：**能用接口声明就别用具体类**。

## 🆕 不可变集合（Java 9+）

### `List.of / Set.of / Map.of`

```java
// 不可变 List（Java 9+）
List<String> list = List.of("a", "b", "c");
list.add("d");    // ❌ 运行时抛 UnsupportedOperationException

// 不可变 Set
Set<String> set = Set.of("a", "b", "c");

// 不可变 Map
Map<String, Integer> map = Map.of("a", 1, "b", 2);
```

**特点**：
- 创建后不能修改
- 更高效（JDK 内部优化）
- **适合常量 / 配置 / API 返回值**

### `Collections.unmodifiableList` —— 包装已有 List

```java
List<String> original = new ArrayList<>();
original.add("a");

List<String> readonly = Collections.unmodifiableList(original);
readonly.add("b");        // ❌ UnsupportedOperationException

// ⚠️ 注意：修改 original 会影响 readonly（只是"只读视图"，不是拷贝）
original.add("c");
System.out.println(readonly);   // [a, c]
```

### `List.copyOf` —— 真不可变拷贝（Java 10+）

```java
List<String> immutable = List.copyOf(original);
// immutable 是独立的副本，修改 original 不影响 immutable
```

**Builder 模式里用的就是这个**：
```java
this.vegetables = List.copyOf(b.vegetables);
```

## 🔧 常用 API 速查

### List

```java
list.add(e);              // 末尾添加
list.add(i, e);           // 指定位置添加
list.get(i);              // 按下标取
list.set(i, e);           // 替换
list.remove(i);           // 按下标删
list.remove(e);           // 按值删第一个
list.size();              // 数量
list.isEmpty();           // 是否空
list.contains(e);         // 是否包含
list.indexOf(e);          // 查索引
list.clear();             // 清空
list.subList(from, to);   // 子列表视图
```

### Set

```java
set.add(e);            // 添加（重复返回 false）
set.remove(e);         // 删除
set.contains(e);       // 判断
set.size();
set.isEmpty();
```

### Map

```java
map.put(k, v);                    // 放入
map.get(k);                       // 取出
map.getOrDefault(k, defaultV);    // 取不到用默认
map.containsKey(k);
map.containsValue(v);
map.remove(k);
map.size();
map.keySet();                     // 所有 key
map.values();                     // 所有 value
map.entrySet();                   // 所有键值对
map.forEach((k, v) -> {});        // Java 8+ 遍历
map.computeIfAbsent(k, key -> new ArrayList<>()).add(x);    // 很实用
```

### Stream（Java 8+）

```java
list.stream()
    .filter(s -> s.length() > 3)
    .map(String::toUpperCase)
    .collect(Collectors.toList());

// Java 16+ 简化
list.stream().filter(...).map(...).toList();
```

## 🎯 选型指南

### "我要存一堆东西，选什么？"

| 需求 | 推荐 |
|------|------|
| 有序、可重复、普通场景 | **ArrayList** |
| 有序、可重复、多读少写、并发 | `CopyOnWriteArrayList` |
| 不重复、顺序不重要 | **HashSet** |
| 不重复、保持插入顺序 | `LinkedHashSet` |
| 不重复、自动排序 | `TreeSet` |
| 键值对、普通场景 | **HashMap** |
| 键值对、保持插入顺序（如 LRU）| `LinkedHashMap` |
| 键值对、按 key 排序 | `TreeMap` |
| 键值对、多线程并发 | `ConcurrentHashMap` |

## ⚠️ 常见误区

### 误区 1：`ArrayList` vs `Array`
- `Array`（`int[]`, `String[]`）：固定大小，原始数组
- `ArrayList`：动态大小，基于数组实现的 List

### 误区 2：遍历时删除
```java
for (String s : list) {
    if (...) list.remove(s);     // ❌ 抛 ConcurrentModificationException
}

// 正确方式
list.removeIf(s -> ...);          // Java 8+
// 或：
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (...) it.remove();
}
```

### 误区 3：`HashMap` 的 key 改了会怎样？
```java
Map<User, Integer> map = new HashMap<>();
User u = new User("Alice");
map.put(u, 1);

u.setName("Bob");                 // 🚨 key 的 hashCode 变了
map.get(u);                       // 可能返回 null！（hash 找不到对应桶）
```

**规则**：HashMap 的 key 应该是**不可变**的（或 hash 不变）。典型用 `String`、`Integer` 做 key。

### 误区 4：`Arrays.asList(arr)` 返回的不是 ArrayList
```java
List<String> list = Arrays.asList("a", "b", "c");
list.add("d");     // ❌ UnsupportedOperationException
```

`Arrays.asList` 返回的是内部类 `Arrays$ArrayList`，**固定大小**（只能 set，不能 add/remove）。

### 误区 5：`Map.of` 不允许 null
```java
Map.of("key", null);    // ❌ NullPointerException
```

Java 9+ 的 `of()` 系列对 null 零容忍。想存 null value 用 `HashMap`。

## 🎁 高频实战技巧

### 1. 集合间互转

```java
// List → Set（去重）
Set<String> set = new HashSet<>(list);

// Set → List
List<String> list = new ArrayList<>(set);

// Array → List
List<String> list = Arrays.asList("a", "b");
List<String> list2 = List.of("a", "b");         // 不可变
List<String> list3 = new ArrayList<>(Arrays.asList("a", "b"));   // 可修改

// List → Array
String[] arr = list.toArray(new String[0]);
```

### 2. Map 的分组统计

```java
Map<String, List<User>> byCity = users.stream()
    .collect(Collectors.groupingBy(User::getCity));
```

### 3. 判断空集合

```java
if (list == null || list.isEmpty()) { ... }
// 或用 Apache Commons
CollectionUtils.isEmpty(list);
```

### 4. `computeIfAbsent` 极其好用

```java
Map<String, List<Order>> byUser = new HashMap<>();

// 老写法
for (Order o : orders) {
    List<Order> userOrders = byUser.get(o.getUserId());
    if (userOrders == null) {
        userOrders = new ArrayList<>();
        byUser.put(o.getUserId(), userOrders);
    }
    userOrders.add(o);
}

// 新写法
for (Order o : orders) {
    byUser.computeIfAbsent(o.getUserId(), k -> new ArrayList<>()).add(o);
}
```

## 📌 一句话总结

> **Java 集合 = 接口定义能力（List/Set/Map）+ 实现类定义性能（ArrayList/HashMap 等）**。**泛型**让你在编译期管住类型，**菱形运算符**省打字。**声明用接口，实例化用具体类**，是代码易维护的关键。
