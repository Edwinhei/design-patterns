## 📖 这本笔记的使用方式

### 什么时候看？
- 📍 **正在学某一课**：看该课对应的 Java 笔记，帮助你看懂代码
- 📍 **过几天忘了**：直接翻对应知识点复习
- 📍 **面试前**：这些都是真·面试高频题（volatile、synchronized、类加载……）

### 怎么看？
- 每篇**自成体系**，不依赖前后文
- 结构统一：**场景问题 → 原理解释 → 代码示例 → 常见误区**
- 标注**"从哪一课学到"**，方便回溯上下文

## 🎯 学习原则

### 1. **不孤立学 Java，跟着问题学**
每条知识都是为了解决设计模式里的某个"为什么"。**背离场景学 Java 是最低效的方式**。

### 2. **理解优先于记忆**
能自己画出时序图 / 内存图，比背下定义重要 10 倍。

### 3. **见一个沉淀一个**
不要等到"什么时候找时间系统学一下 Java" —— 你永远不会有那个时间。**当下撞上就当下搞懂+记下来**。

### 4. **不贪多**
遇到与当前无关的深入主题（如 JIT 编译、CPU 缓存一致性协议）先记一笔，**等真正用到再展开**。
